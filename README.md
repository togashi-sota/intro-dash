# ＝LOVE イントロクイズ（ポートフォリオ版）

＝LOVE（イコラブ）の楽曲をテーマにした、ブラウザで遊べるイントロ当てクイズです。フレームワークを使わない素のHTML / CSS / JavaScriptで、個人のポートフォリオ作品として開発しています。

> **本リポジトリで公開している曲データ・音源はすべてダミー（仮のデータ）です。**
> 著作権保護のため、実際の＝LOVEの楽曲音源は含めていません。詳しくは「[音源・著作権について](#音源著作権について)」をご覧ください。

## このアプリでできること

- スタート画面で「出題数」（5問 / 10問 / 全曲）と「カテゴリ」（表題曲のみ / 全曲）を選んでスタート
- イントロ音源を聞いて、4択から曲名を回答
- 1問あたり10秒の制限時間内に、早く正解するほど高得点（スピードボーナス方式）
- 全問終了後に、合計得点と1問ごとの正誤内訳を表示

## 工夫した点

- 状態管理を1つの`gameState`オブジェクトに集約し、画面切り替え（`screens.js`）・出題ロジック（`quiz.js`）・タイマー（`timer.js`）・得点計算（`score.js`）・音声再生（`audio.js`）を役割ごとに分離した設計
- 選択肢クリックとタイマーの時間切れがほぼ同時に発生しても、二重に得点処理されないようガード処理を実装
- 音源の読み込み・再生に失敗した場合も、エラーメッセージを表示するだけでアプリ全体は停止しない設計
- カテゴリで絞り込んだ結果、出題可能な曲が4曲未満になる場合はスタートさせずにエラー表示するバリデーション
- スマホ幅（520px以下）・非常に狭い画面幅（360px以下）に対応したレスポンシブレイアウト
- OSの「モーションを減らす」設定（`prefers-reduced-motion`）に対応したアニメーション制御
- 曲データを`js/data/songs.js`の配列1箇所に集約しており、将来的に曲数を増やす場合もこのファイルへの追記だけで対応できる設計

## 技術構成

- HTML / CSS / JavaScript（フレームワーク・ライブラリ不使用）
- JavaScriptはES Modules（`import` / `export`）でファイルを分割
- Google Fonts（M PLUS Rounded 1c）

## ローカルでの起動方法

このアプリはES Modulesを使用しているため、`index.html`をダブルクリックして直接開いても動作しません（ブラウザのセキュリティ制限のため）。簡易的なローカルサーバーを経由して開いてください。

**VSCodeの「Live Server」拡張機能を使う場合**

1. VSCodeでこのフォルダを開く
2. `index.html`を右クリック →「Open with Live Server」を選択

## 音源・著作権について

本アプリは＝LOVEのファンによる非公式の個人開発プロジェクトです。公開している本リポジトリには、著作権保護のため実際の楽曲音源は含めていません。`assets/audio/dummy/`に置かれたダミー音源（動作確認用の電子音）のみを使用しています。

実際の＝LOVE楽曲音源を試したい場合は、`assets/audio/local/`（`.gitignore`で除外済み・リポジトリには含まれません）にローカル環境でのみ音源ファイルを配置する設計になっており、個人利用の範囲を超えて音源を公開・配布することはありません。
